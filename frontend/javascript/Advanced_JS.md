Этот раздел предназначен для понимания внутренней механики JavaScript, что критично для разработчиков уровня Middle и выше.

---

## 1. Event Loop и асинхронность ⏳

JavaScript — однопоточный язык, но он умеет выполнять асинхронные задачи благодаря **Event Loop** (Цикл событий).

### Как это работает (Call Stack & Task Queues)
1. **Call Stack (Стек вызовов):** Место, где хранятся текущие выполняемые функции (LIFO).
2. **Web APIs (в браузере):** Фоновые задачи (setTimeout, fetch, DOM events).
3. **Microtask Queue (Очередь микрозадач):** Сюда попадают коллбэки от `Promise`, `MutationObserver`, `queueMicrotask`. **Имеет высший приоритет.**
4. **Task Queue / Macrotask Queue (Очередь макрозадач):** Сюда попадают `setTimeout`, `setInterval`, `setImmediate`, I/O задачи.

> **Важно:** Event Loop не перейдет к макрозадачам, пока очередь микрозадач не станет пустой.

#### Пример визуализации приоритетов:
```javascript
console.log('1'); // Синхронно

setTimeout(() => console.log('2'), 0); // Макрозадача

Promise.resolve().then(() => console.log('3')); // Микрозадача

console.log('4'); // Синхронно

// Вывод: 1, 4, 3, 2
```

## 2. Замыкания (Closures) 🔒

**Замыкание** — это функция вместе со всеми внешними переменными, которые ей доступны. В JS замыкание создается каждый раз при создании функции.

### Практическое применение: Инкапсуляция
```javascript
function createCounter() {
  let count = 0; // Приватная переменная
  return {
    increment() { count++; return count; },
    decrement() { count--; return count; },
    getCount() { return count; }
  };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.count); // undefined (переменная скрыта)
```

## 3. Прототипы и наследование 🧬

В JS нет классов в классическом понимании (как в Java). Классы в ES6 — это "синтаксический сахар" над прототипами.

### Prototype Chain (Цепочка прототипов)
Каждый объект имеет скрытое свойство `[[Prototype]]` (доступно через `__proto__` или `Object.getPrototypeOf()`), которое ссылается на другой объект.

```javascript
const animal = { eats: true };
const rabbit = Object.create(animal); // rabbit.__proto__ === animal

console.log(rabbit.eats); // true (взято из прототипа)
```

## 4. Контекст `this` и потеря контекста 🎯

Значение `this` определяется в момент **вызова** функции, а не её объявления (кроме стрелочных функций).

| Способ вызова | Значение `this` |
| :--- | :--- |
| Обычный вызов | `window` (или `undefined` в strict mode) |
| Метод объекта | Сам объект |
| Конструктор (`new`) | Новый созданный объект |
| Стрелочная функция | Берет `this` из внешнего лексического окружения |

### Явная привязка: `call`, `apply`, `bind`
- `call(thisArg, arg1, arg2)` — вызывает функцию сразу.
- `apply(thisArg, [argsArray])` — вызывает функцию сразу, аргументы массивом.
- `bind(thisArg)` — возвращает **новую** функцию с привязанным контекстом.

## 5. Память и Garbage Collection (GC) 🧹

JS использует автоматическое управление памятью. Основной алгоритм — **Mark-and-Sweep** (Пометь и очисти).

### Основные причины утечек памяти:
1. **Забытые глобальные переменные:** `function f() { leak = "I am global"; }`
2. **Забытые таймеры или интервалы:** `setInterval` внутри компонентов, которые не очищаются.
3. **Замыкания:** Если большая переменная остается в области видимости долгоживущей функции.
4. **Удаленные DOM-узлы:** Ссылки на DOM-элементы в JS-объектах после удаления их со страницы.
