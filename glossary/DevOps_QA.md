# 🛠️ DevOps & Testing Terms

| Английский (Term) | Русский (Translation) | Описание (Short Explanation) |
| :--- | :--- | :--- |
| **Artifact** | Артефакт | Файл, готовый к запуску (например, .jar или .exe). |
| **CI (Continuous Integration)** | Непрерывная интеграция | Автоматическая сборка и тестирование кода при каждом изменении. |
| **CD (Continuous Delivery)** | Непрерывная поставка | Автоматическая подготовка изменений к выпуску. |
| **Cluster** | Кластер | Группа серверов, работающих как единая система. |
| **Container** | Контейнер | Изолированная среда для запуска приложения со всеми зависимостями (Docker). |
| **Deployment** | Деплой (развертывание) | Процесс переноса кода на рабочий сервер. |
| **Environment** | Окружение | Место, где запущен код (Development, Staging, Production). |
| **HPA (Horizontal Pod Autoscaler)** | Автомасштабирование | Механизм, который меняет число запущенных серверов в зависимости от нагрузки. |
| **Observability** | Наблюдаемость | Система мониторинга (логи, метрики, трассировки). |
| **Regression Testing** | Регрессионное тестирование | Проверка того, что новые изменения не сломали старый функционал. |
| **Sanity Check** | Проверка на адекватность | Быстрый тест самого важного функционала приложения. |
| **Staging** | Стейджинг | Тестовый сервер, максимально похожий на реальный (боевой). |
| **Unit Test** | Модульный тест | Тест одной маленькой части кода (функции или класса) в изоляции. |
