# Docker 🐳

Docker позволяет «упаковать» приложение вместе со всеми зависимостями в один контейнер. Подробный разбор и примеры — в `Docker.md`.

## Содержание

1.  **[Подробный гайд по Docker: Dockerfile и Docker Compose](./Docker.md)** — Полный конспект с примерами для Spring Boot.

### Краткий справочник

| Команда | Описание |
| :--- | :--- |
| `docker build -t my-app .` | Собрать образ из `Dockerfile` в текущей папке. |
| `docker run -p 8080:8080 my-app` | Запустить контейнер из образа. |
| `docker ps` | Список запущенных контейнеров. |
| `docker stop <id>` | Остановить контейнер. |
| `docker images` | Список локальных образов. |
| `docker rmi <image>` | Удалить образ. |
| `docker logs -f <id>` | Читать логи контейнера в реальном времени. |
| `docker exec -it <id> sh` | Войти внутрь работающего контейнера. |
| `docker-compose up -d` | Поднять все сервисы из `docker-compose.yml`. |
| `docker-compose down` | Остановить и удалить все сервисы. |
| `docker-compose up --build` | Пересобрать образы и поднять. |

### Ключевые концепции
* **Image (Образ)** — Шаблон (read-only), слоистый файловая система. Собирается из `Dockerfile`.
* **Container (Контейнер)** — Запущенный экземпляр образа (read-write слой поверх образа).
* **Volume** — Постоянное хранилище данных вне контейнера (для БД, чтобы данные не удалялись).
* **Networks** — Изолированные сети для связи между контейнерами внутри Compose.
* **`.dockerignore`** — Аналог `.gitignore` для исключения файлов из сборки образа.
