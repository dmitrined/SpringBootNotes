# React Architecture: Под капотом ⚙️

Понимание внутренней работы React помогает писать производительный код и избегать лишних ре-рендеров.

---

## 1. Virtual DOM и Алгоритм согласования (Reconciliation) 🌳

**Virtual DOM** — это легковесная копия реального DOM в виде JavaScript-объектов.

### Как работает обновление (Процесс Diffing):
1. При изменении `state` или `props` React создает новое дерево Virtual DOM.
2. React сравнивает (diff) новое дерево со старым.
3. Находит минимальное количество изменений.
4. Применяет эти изменения к реальному DOM (Batching).

### Правила Diffing:
- **Разные типы элементов:** Если тип элемента изменился (например, был `<div>`, стал `<span>`), React полностью уничтожает старое дерево и строит новое.
- **Ключи (Keys) в списках:** `key` помогает React идентифицировать элементы. Если ключ не изменился, React просто перемещает элемент, а не пересоздает его. *Никогда не используйте index массива в качестве ключа, если список может меняться!*

## 2. Жизненный цикл и Фазы 🔄

В React 16+ выделяют три основные фазы:

1. **Render phase:** Вычисление изменений (чистая фаза, может прерываться).
2. **Pre-commit phase:** Чтение DOM перед изменениями (редко используется, например, `getSnapshotBeforeUpdate`).
3. **Commit phase:** Применение изменений к реальному DOM и запуск сайд-эффектов (`componentDidMount`, `useEffect`).

## 3. Fiber Architecture 🧶

**React Fiber** — это текущий движок React (с версии 16.0). Его главная цель — сделать рендеринг инкрементальным.

- **Зачем это нужно?** Раньше (Stack Reconciler) рендеринг был синхронным. Если дерево было огромным, браузер "зависал", пока React не закончит.
- **Что делает Fiber?** Он разбивает работу на мелкие порции (фиберы) и может приостанавливать рендеринг, чтобы дать браузеру отрисовать анимацию или обработать ввод пользователя, а затем вернуться к работе.

## 4. State Management: Как выбрать? 🧠

| Инструмент | Когда использовать | Плюсы |
| :--- | :--- | :--- |
| **Local State (`useState`)** | Данные нужны только одному компоненту или его детям. | Простота, инкапсуляция. |
| **Context API** | Редко меняющиеся данные (тема, язык, данные пользователя). | Нет лишних зависимостей. |
| **Zustand** | Большинство современных приложений. Глобальный стейт. | Маленький размер, отличный DX. |
| **Redux Toolkit** | Огромные корпоративные приложения со сложной логикой. | Строгая структура, мощный DevTools. |
| **React Query / SWR** | Кэширование данных с сервера (Server State). | Авто-обновление, обработка ошибок/загрузки. |

## 5. Оптимизация производительности 🚀

1. **Избегайте анонимных функций в props:** `onClick={() => doSomething()}` создает новую функцию при каждом рендере. Используйте `useCallback`.
2. **Мемоизация компонентов:** Оборачивайте тяжелые компоненты в `React.memo`.
3. **Оптимизация контекста:** Разбивайте один большой контекст на несколько маленьких, чтобы избежать ре-рендера всего приложения при изменении одного поля.
4. **Code Splitting:** Разделяйте бандл на части с помощью `React.lazy` и `Suspense`.
