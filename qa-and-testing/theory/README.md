### 2. Виды и методы тестирования

| По объекту | По запуску | По доступу к коду (Методы) |
| :--- | :--- | :--- |
| **Функциональное** (что делает) | **Ручное** (Manual) | **[Black Box](./Testing_Methods.md)** (не видим код) |
| **Нефункциональное** (нагрузка, UI) | **Автоматизированное** | **[White Box](./Testing_Methods.md)** (видим код/структуру) |
| | | **[Gray Box](./Testing_Methods.md)** (частичный доступ) |

### 3. Пирамида тестирования (Testing Pyramid)
Концепция, говорящая о том, что тестов низкого уровня должно быть больше.
- **Unit Tests** (Основание) — Тестирование отдельных методов. Быстрые, дешевые.
- **Integration Tests** — Проверка взаимодействия модулей/БД.
- **E2E (End-to-End) Tests** (Вершина) — Прохождение сценария как пользователь. Медленные, дорогие.

### 4. Жизненный цикл бага (Bug Lifecycle)
`New` -> `Assigned` -> `In Progress` -> `Fixed` -> `Pending Retest` -> `Verified` -> `Closed`.

### 5. [Стратегии тестирования](./Testing_Strategy.md) 🏗️
- Подходы: Shift-Left, Shift-Right, TDD, BDD, ATDD.
- Архитектуры: Пирамида vs Соты (Honeycomb).
