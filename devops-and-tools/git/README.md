# Git 🌿

Git — стандарт де-факто для управления версиями кода. Это не просто инструмент сохранения файлов — это система для совместной работы команды и история всех изменений.

## Содержание

### Основные понятия
* **Repository (Репозиторий)** — Папка под управлением Git (локальная и удалённая на GitHub/GitLab).
* **Commit** — Сохранённый снимок состояния файлов. Неизменяем. Имеет уникальный SHA-хэш.
* **Branch (Ветка)** — Независимая линия разработки. Дешёвый способ изолировать работу.
* **Working Tree → Index (Staging) → HEAD** — Три состояния файла в Git.

### Базовые команды

| Команда | Описание |
| :--- | :--- |
| `git init` | Создать новый локальный репозиторий. |
| `git clone <url>` | Скопировать удалённый репозиторий. |
| `git status` | Показать статус файлов (изменены, staged, untracked). |
| `git add <file>` / `git add .` | Добавить файл(ы) в staging area. |
| `git commit -m "msg"` | Зафиксировать изменения из staging area. |
| `git log --oneline --graph` | Компактная история коммитов с деревом. |
| `git diff` | Посмотреть различия в незафиксированных файлах. |

### Работа с ветками и слиянием

| Команда | Описание |
| :--- | :--- |
| `git branch <name>` | Создать ветку. |
| `git checkout -b <name>` | Создать и переключиться на ветку (или `git switch -c`). |
| `git merge <branch>` | Слить ветку в текущую (создаёт merge commit). |
| `git rebase <branch>` | Перенести коммиты текущей ветки на вершину другой (история "чище"). |
| `git cherry-pick <SHA>` | Применить конкретный коммит в текущую ветку. |

### Отмена изменений

| Команда | Описание |
| :--- | :--- |
| `git restore <file>` | Отменить изменения в рабочей директории. |
| `git restore --staged <file>` | Убрать файл из staging area. |
| `git revert <SHA>` | Создать **новый** коммит, отменяющий указанный (безопасно для истории). |
| `git reset --hard <SHA>` | ⚠️ Опасно! Сдвинуть HEAD назад, удалив изменения. |

### Работа с удалённым репозиторием

| Команда | Описание |
| :--- | :--- |
| `git fetch` | Скачать изменения, но **не** применять. |
| `git pull` | Скачать и применить (`fetch` + `merge`). |
| `git push origin <branch>` | Отправить ветку на GitHub/GitLab. |
| `git push --force-with-lease` | Принудительный push (безопаснее `--force`). |

### Полезные инструменты

* **`git stash`** — Временно спрятать незакоммиченные изменения.
* **`git stash pop`** — Вернуть их обратно.
* **`git tag -a v1.0.0 -m "Release"`** — Создать аннотированный тег для релиза.
* **`.gitignore`** — Файл с правилами исключения файлов из отслеживания (напр., `node_modules/`, `*.jar`, `.env`).

### Git Flow (Модель ветвления)
Популярная стратегия для командной работы:
* **`main`** — Только Production-ready код.
* **`develop`** — Основная ветка разработки.
* **`feature/...`** — Отдельная ветка для каждой фичи (ответвляется от `develop`).
* **`release/...`** — Финальная подготовка к релизу (стабилизация).
* **`hotfix/...`** — Срочные исправления прямо из `main`.
