# HTTP Methods (Методы запроса) 🔑

HTTP-методы (их часто называют "глаголами") определяют, какое действие клиент хочет совершить с ресурсом. В Spring Boot для каждого метода есть своя аннотация в контроллере.

---

## 📝 Основные методы

| Метод | Аннотация | Идемпотентность? | Описание |
| :--- | :--- | :--- | :--- |
| **GET** | `@GetMapping` | ✅ Да | **Чтение**. Запрашивает данные. Не должен менять ничего на сервере. |
| **POST** | `@PostMapping` | ❌ Нет | **Создание**. Отправляет данные для создания нового ресурса. |
| **PUT** | `@PutMapping` | ✅ Да | **Замена**. Полностью заменяет существующий ресурс новыми данными. |
| **PATCH** | `@PatchMapping` | ❌ Нет* | **Частичное обновление**. Меняет только отдельные поля ресурса. |
| **DELETE** | `@DeleteMapping` | ✅ Да | **Удаление**. Удаляет указанный ресурс. |

---

## 🧠 Ключевые понятия

### 🛡 Безопасность (Safe)
Метод считается **безопасным**, если он только читает данные и никак не меняет состояние системы.
*   **Безопасный**: `GET`, `HEAD`, `OPTIONS`.
*   **Небезопасный**: `POST`, `PUT`, `DELETE`, `PATCH`.

### 🔄 Идемпотентность (Idempotent)
Метод **идемпотентен**, если повторный вызов одного и того же запроса дает тот же результат, что и первый вызов (состояние сервера не меняется дальше).

> [!TIP]
> **Пример:**
> - `DELETE /users/1` — первый раз удалит юзера. Второй, третий и сотый разы — юзера уже нет, ничего не изменится. Это **идемпотентно**.
> - `POST /orders` — каждый клик создает новый заказ. Пять кликов = пять заказов. Это **НЕ идемпотентно**.

## 💻 Использование в Spring Boot

```java
@RestController
@RequestMapping("/api/products")
public class ProductController {

    // 1. GET — Получить список или один товар
    @GetMapping("/{id}")
    public Product get(@PathVariable Long id) { ... }

    // 2. POST — Создать новый товар
    @PostMapping
    public Product create(@RequestBody Product product) { ... }

    // 3. PUT — Обновить товар целиком (заменить)
    @PutMapping("/{id}")
    public Product update(@PathVariable Long id, @RequestBody Product product) { ... }

    // 4. PATCH — Обновить только цену товара
    @PatchMapping("/{id}")
    public Product updatePrice(@PathVariable Long id, @RequestBody Map<String, Object> updates) { ... }

    // 5. DELETE — Удалить товар
    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) { ... }
}
```

## ❓ Часто задаваемые вопросы

### PUT vs PATCH: В чем разница?
- **PUT**: Вы присылаете объект целиком. Если вы не пришлете какое-то поле, оно в базе может стать `null` (зависит от логики). Это как "перезаписать файл".
- **PATCH**: Вы присылаете только те поля, которые хотите изменить. Остальные остаются прежними. Это как "правка в строке".

### Почему POST не идемпотентен?
Потому что основной смысл `POST` — это действие, которое порождает изменения при каждом вызове (создание записи, отправка email, проведение транзакции).

### Можно ли передавать Body в GET?
Технически — да, но по стандартам HTTP это **крайне не рекомендуется**. Большинство серверов и кэшей просто проигнорируют тело в `GET`-запросе. Для фильтрации используйте `@RequestParam`.
