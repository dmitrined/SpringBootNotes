# Framework: React & Next.js ⚛️

Современные инструменты для построения сложных пользовательских интерфейсов. React — самая популярная UI-библиотека, Next.js — мощный фреймворк над ней.

## Содержание

### React

#### Компоненты (Components)
* **Function Components** — Современный стандарт. Компонент = функция, возвращающая JSX.
* **JSX** — XML-подобный синтаксис в JS. Компилируется в `React.createElement()`.
* **Props** — Входные параметры компонента (только читаем, передаются сверху вниз).
* **State** — Внутреннее состояние компонента. Изменение стейта = перерисовка.
* **Lifting State Up** — Поднятие стейта к общему предку для обмена данными между компонентами.

#### Хуки (Hooks)
* **`useState`** — Управление локальным состоянием. `const [count, setCount] = useState(0)`.
* **`useEffect`** — Сайд-эффекты (fetch данных, подписки). `deps array` — контроль запуска.
* **`useRef`** — Ссылка на DOM-элемент или хранение мутабельного значения без ре-рендера.
* **`useMemo`** — Мемоизация дорогого вычисления (кэшировать значение).
* **`useCallback`** — Мемоизация функции (передавать стабильную ссылку в дочерние компоненты).
* **`useContext`** — Читать значение из React Context без prop drilling.
* **`useReducer`** — Управление сложным состоянием с логикой (аналог Redux для локального стейта).
* **Custom Hooks** — Инкапсуляция повторяемой логики в переиспользуемые функции (`useFetch`, `useForm`).

#### Управление Состоянием (State Management)
* **Context API** — Встроенный способ передачи данных вглубь дерева без prop drilling. Подходит для простых случаев.
* **Zustand** — Легковесная библиотека для глобального стейта, минималистичная альтернатива Redux.
* **Redux Toolkit** — Мощное решение для сложного глобального стейта (`createSlice`, `createAsyncThunk`).

#### Производительность
* **`React.memo`** — Предотвращает ре-рендер компонента, если его props не изменились.
* **`key` prop** — Почему в списках (`map`) НУЖЕН уникальный `key`, и как он помогает алгоритму согласования (Reconciliation).
* **Code Splitting & Lazy Loading** — `React.lazy()` + `<Suspense>`.

#### Маршрутизация (React Router)
* `<BrowserRouter>`, `<Routes>`, `<Route>`, `<Link>`, `<NavLink>`.
* **Программная навигация** — `useNavigate()`.
* **Параметры маршрута** — `useParams()`, `useSearchParams()`.

### Next.js

#### Ключевые концепции
* **App Router (Next.js 13+)** — Файловая система как маршруты (через папку `app`).
* **Server Components vs Client Components** — По умолчанию все компоненты серверные, `"use client"` для клиентских.
* **Стратегии рендеринга**:
  - **SSR (Server Side Rendering)** — Страница генерируется на сервере при каждом запросе.
  - **SSG (Static Site Generation)** — Страница генерируется при сборке.
  - **ISR (Incremental Static Regeneration)** — Статика с периодическим обновлением.
* **Server Actions** — Выполнение кода на сервере прямо из форм (`"use server"`).
* **Data Fetching** — Нативный `fetch` с кэшированием (`cache: 'no-store'`, `revalidate`).

#### Файловая структура (App Router)
* `layout.tsx` — Общий лейаут (оберточный компонент).
* `page.tsx` — Страница маршрута.
* `loading.tsx` — UI в состоянии загрузки.
* `error.tsx` — UI при ошибке.
* `not-found.tsx` — 404 страница.
* `route.ts` — API Routes (бэкенд-эндпоинты прямо в Next.js).
