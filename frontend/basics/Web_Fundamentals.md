# Web Fundamentals: Как работает веб 🌐

Понимание того, как браузер взаимодействует с сервером и отрисовывает страницу, отличает профессионального фронтенд-разработчика от новичка.

---

## 1. Протокол HTTP / HTTPS 🛰️

**HTTP (HyperText Transfer Protocol)** — основной протокол передачи данных в вебе.

### Ключевые понятия:
- **Методы запроса:** `GET` (получение), `POST` (создание), `PUT` (обновление целиком), `PATCH` (частичное обновление), `DELETE` (удаление).
- **Заголовки (Headers):** `Content-Type`, `Authorization` (Bearer token), `Accept`, `Cache-Control`, `User-Agent`.
- **Статусы ответов:**
    - `2xx` — Успех (`200 OK`, `201 Created`).
    - `3xx` — Перенаправление (`301 Moved Permanently`, `304 Not Modified`).
    - `4xx` — Ошибка клиента (`400 Bad Request`, `401 Unauthorized`, `404 Not Found`).
    - `5xx` — Ошибка сервера (`500 Internal Server Error`).

## 2. Критический путь рендеринга (Critical Rendering Path) 🏎️

Процесс превращения HTML, CSS и JS в пиксели на экране.

1. **DOM (Document Object Model):** Браузер парсит HTML и строит дерево узлов.
2. **CSSOM (CSS Object Model):** Браузер парсит CSS и строит дерево стилей.
3. **Render Tree:** Объединение DOM и CSSOM (только видимые элементы).
4. **Layout (Flow):** Расчет геометрии (позиция и размер каждого элемента).
5. **Paint:** Отрисовка текста, цветов, границ, теней.
6. **Composite:** Сборка слоев (выполняется на GPU).

> **Совет по оптимизации:** Свойства `transform` и `opacity` не вызывают *Layout* и *Paint*, они работают сразу на этапе *Composite*, что делает анимации плавными (60 FPS).

## 3. DOM, BOM и хранилища 📦

- **DOM:** Интерфейс для управления структурой и содержимым документа.
- **BOM (Browser Object Model):** Объекты для управления браузером: `window`, `navigator`, `location`, `history`, `screen`.

### Хранение данных в браузере:
| Тип | Размер | Время жизни | Доступность |
| :--- | :--- | :--- | :--- |
| **LocalStorage** | ~5-10MB | До удаления вручную | Между вкладками (тот же домен) |
| **SessionStorage** | ~5MB | До закрытия вкладки | Только в текущей вкладке |
| **Cookies** | ~4KB | До истечения (Expires) | Отправляются с каждым запросом |
| **IndexedDB** | ГБ+ | Постоянно | Сложные объекты, транзакции |

---

## 4. Безопасность (Frontend Security) 🛡️

1. **XSS (Cross-Site Scripting):** Инъекция вредоносного JS. *Защита:* Экранирование данных, использование `CSP` (Content Security Policy).
2. **CSRF (Cross-Site Request Forgery):** Подделка запроса от имени пользователя. *Защита:* Использование `CSRF-токенов`, `SameSite` куки.
3. **CORS (Cross-Origin Resource Sharing):** Механизм, позволяющий серверу указать, каким доменам разрешен доступ к ресурсам.
