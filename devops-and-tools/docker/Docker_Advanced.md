# Docker Advanced: Оптимизация и лучшие практики 🐳

Для уровня Middle важно не просто уметь запускать контейнеры, но и делать их легкими, безопасными и производительными.

---

## 1. Multi-stage Builds: Уменьшение размера образа 📉

Это техника сборки, при которой мы используем один образ для компиляции приложения, а другой (гораздо меньше) — для его запуска.

### Пример для Spring Boot:
```dockerfile
# Stage 1: Сборка (Build)
FROM maven:3.8.5-openjdk-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# Stage 2: Запуск (Run)
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```
*Результат:* Образ весит ~200МБ вместо 800МБ+, так как в нем нет Maven и исходного кода.

## 2. Слоеная архитектура и кэширование 🍰

Docker кэширует каждый слой (команду в Dockerfile). Если слой не изменился, Docker берет его из кэша.

**Плохо:**
```dockerfile
COPY . .
RUN mvn install
```
*Почему?* Любое изменение в коде сбросит кэш для `RUN mvn install`, и зависимости будут качаться заново.

**Хорошо:**
```dockerfile
COPY pom.xml .
RUN mvn dependency:go-offline
COPY src ./src
RUN mvn package
```
*Почему?* Зависимости скачаются один раз и закешируются. Они перекачаются только если изменится `pom.xml`.

## 3. Docker Compose: Сложные окружения 🏗️

Compose позволяет описать всю инфраструктуру (БД, Кэш, Бэкенд, Фронтенд) в одном файле.

### Полезные фишки:
- **depends_on:** Порядок запуска (но не готовности! Для готовности используйте `healthcheck`).
- **networks:** Изоляция сервисов. Бэкенд может видеть БД, а Фронтенд — нет.
- **volumes:** Прокидывание конфигов или сохранение данных БД.
- **env_file:** Вынос секретов в отдельный `.env` файл.

## 4. Безопасность (Docker Security) 🛡️

1. **Не запускайте от root:** В Dockerfile используйте `USER appuser`.
2. **Используйте официальные образы:** Ищите пометку `Official Image` на Docker Hub.
3. **Сканируйте на уязвимости:** Команда `docker scan <image_name>`.
4. **Ограничивайте ресурсы:** В Compose можно задать `deploy.resources.limits` (CPU, RAM).

## 5. Полезные инструменты 🛠️

- **Dive:** Инструмент для анализа слоев Docker-образа и поиска лишнего мусора.
- **Hadolint:** Линтер для Dockerfile (проверяет на ошибки и плохие практики).
- **Portainer:** Графический интерфейс для управления контейнерами.
