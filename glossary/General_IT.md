# 🌐 Общие IT-термины (General Terms)

| Английский (Term) | Русский (Translation) | Описание (Short Explanation) |
| :--- | :--- | :--- |
| **Boilerplate** | Шаблонный код | Код, который нужно писать везде почти без изменений (например, геттеры/сеттеры). |
| **Best Practices** | Лучшие практики | Наиболее эффективные и проверенные способы решения задач. |
| **Legacy Code** | Унаследованный код | Старый код, который сложно поддерживать, но он все еще работает. |
| **Refactoring** | Рефакторинг | Переработка кода для улучшения структуры без изменения функциональности. |
| **Tech Debt** | Техдолг | Накопленные ошибки или упрощения в коде, которые придется исправлять в будущем. |
| **Proof of Concept (PoC)** | Доказательство концепции | Черновик или прототип, чтобы проверить, работает ли идея. |
| **Scalability** | Масштабируемость | Способность системы справляться с растущей нагрузкой. |
| **Throughput** | Пропускная способность | Количество запросов, которые система может обработать за единицу времени. |
| **Latency** | Задержка | Время, которое требуется для обработки одного запроса. |
| **Constraint** | Ограничение | Условие или рамка, внутри которой работает система. |
| **Concurrency** | Конкурентность | Способность выполнять несколько задач одновременно (логически). |
| **Parallelism** | Параллелизм | Физически одновременное выполнение нескольких задач. |
| **Trade-off** | Компромисс | Выбор между двумя вариантами, когда улучшение одного параметра ухудшает другой. |
